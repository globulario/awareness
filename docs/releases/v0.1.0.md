# Awareness v0.1.0 — First Standalone Release

**Released:** 2026-05-14

---

## What Is Awareness?

Awareness is a project-aware context, preflight, invariant, bundle, and MCP toolchain for AI-assisted software development.

It gives AI agents working on your project access to your project's:

- **Invariants** — laws the code must uphold (idempotency, context propagation, state determinism)
- **Failure modes** — known ways the system can break and their symptoms
- **Forbidden fixes** — patches that look plausible but are wrong for this project

These are hand-authored YAML files checked into your repository. Awareness makes them machine-readable and actionable.

Awareness is not a Globular component. It is a standalone, project-agnostic module. Globular is one project that uses it.

---

## What v0.1.0 Includes

### Core Engine

- `project.ProjectProfile` — project identity resolution from `.awareness.yaml`
- `project.Doctor` — static health check for the project profile
- `preflight.ClassifyTask` — classifies a task description into categories
- `preflight.RawKnowledgeFallback` — keyword search across YAML knowledge files
- `bundle.BundleManifest` — portable snapshot format for knowledge files
- `bundle.Build` — builds an Awareness bundle from a project profile
- `runtime.Adapter` / `runtime.NullAdapter` — adapter interface and null implementation
- `graph.Node`, `graph.Edge`, `graph.Graph` — graph primitive types (production SQLite engine in services)

### CLI (`awareness`)

```bash
awareness profile doctor [--project-root PATH]
awareness preflight [--changed] [--task "..."] [--format json|text] [--project-root PATH]
awareness bundle build [--project-root PATH] [--out DIR] [--revision SHA]
```

### MCP Server (`awareness-mcp`)

Serves 9 tools over JSON-RPC 2.0 / stdio:

| Tool | Purpose |
|------|---------|
| `awareness_profile_doctor` | Static health check on the project profile |
| `awareness_runtime_status` | Adapter status (runtime_disabled for NullAdapter) |
| `awareness_preflight` | Preflight check for a task or changed files |
| `awareness_context` | Keyword search across knowledge YAML files |
| `awareness_graph_query` | Graph node query (graceful graph_not_available when no graph) |
| `awareness_node_context` | Context for a file path or node ID |
| `awareness_invariant_lookup` | Search invariants by keyword |
| `awareness_failure_mode_lookup` | Search failure modes by keyword |
| `awareness_bundle_inspect` | Inspect a bundle directory |

Works without a Globular cluster. Non-Globular projects get `runtime_disabled`.

### Examples

Four working example projects:

- `examples/go-service` — Go service with context propagation and idempotency invariants
- `examples/typescript-app` — TypeScript app with typed API and immutable state invariants
- `examples/bpmn-engine` — BPMN workflow engine with process determinism invariants
- `examples/mixed-monorepo` — Mixed-language monorepo with backwards compatibility invariants

## Quickstart (10 minutes)

```bash
# 1. Install
go install github.com/globulario/awareness/cmd/awareness@v0.1.0
go install github.com/globulario/awareness/cmd/awareness-mcp@v0.1.0

# 2. Create a project profile
cd my-project
mkdir -p .awareness

cat > .awareness.yaml <<'YAML'
project:
  name: my-project
  kind: generic
runtime:
  enabled: false
  adapter: null
YAML

# 3. Write one invariant
cat > .awareness/invariants.yaml <<'YAML'
invariants:
  - id: no.silent.errors
    title: Errors must not be silently discarded
    description: Every error must be returned, logged, or wrapped. Using _ to discard errors is forbidden.
    severity: critical
    tags: [errors, reliability]
YAML

# 4. Check the profile
awareness profile doctor

# 5. Run preflight
awareness preflight --changed

# 6. Start the MCP server
awareness-mcp --project-root .
```

## MCP Usage (Claude Code)

Add to `.mcp.json` in your project:

```json
{
  "mcpServers": {
    "awareness": {
      "command": "awareness-mcp",
      "args": ["--project-root", "/absolute/path/to/your/project"]
    }
  }
}
```

Claude Code will then have access to all 9 Awareness tools automatically.

See `docs/mcp/claude-code.md` for full configuration guide.

## Bundle Usage

Build a portable bundle for CI, distribution, or agent context:

```bash
awareness bundle build --project-root . --out /tmp/my-project-bundle
cat /tmp/my-project-bundle/bundle.json
```

## CI Usage

```yaml
- name: Run Awareness preflight
  run: |
    go install github.com/globulario/awareness/cmd/awareness@v0.1.0
    awareness preflight --changed --format json > preflight.json
    MATCH_COUNT=$(jq '(.invariants | length) + (.failure_modes | length)' preflight.json)
    if [ "$MATCH_COUNT" -gt 0 ]; then
      echo "Awareness: $MATCH_COUNT knowledge items in scope"
      jq '{invariants, failure_modes}' preflight.json
    fi
```

See `docs/ci/` for complete CI guides.

## Known Limitations

- **Runtime adapters are project-specific.** The NullAdapter ships with standalone Awareness. The GlobularAdapter lives in the services repository. Custom adapters require implementing the `runtime.Adapter` interface.
- **Globular runtime integration lives in services.** Live cluster signals, graph compilation, and runtime state collection require the Globular services-side bridge.
- **Bundle schema is v1 and may evolve.** The `awareness.bundle.v1` format covers the v0.1.0 feature set. Future versions may add fields; readers should handle unknown fields gracefully.
- **MCP tool surface is minimal but growing.** v0.1.0 provides 9 tools. Graph traversal, deeper context, and agent prompt packs are planned for Phase K.
- **Binary release packaging is not yet automated.** Install via `go install`. Pre-built binaries are planned for a future release.
- **The standalone graph engine is stubs only.** The production SQLite-backed graph engine lives in services. The `graph` package provides canonical types; `awareness_graph_query` returns `graph_not_available` unless a `graph.json` snapshot is present in the project's graph cache directory.
